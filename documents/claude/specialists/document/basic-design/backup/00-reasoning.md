# Section 00: Reasoning - Template v1.0
## Construction Management System (施工管理統合システム)

## Objective

Generate Step 0 Reasoning (reasoning.json) for Basic Design document.

This section analyzes SRS to derive components, patterns, and technologies before Basic Design generation.

---

## Technology Context

| Aspect | Value |
|--------|-------|
| **Project** | Construction Management System |
| **Domain** | Civil Engineering / Tunnel Construction |
| **Focus** | Derive components/patterns from requirements |
| **Output** | JSON format (reasoning.json) |

---

## Constraints

### MUST Include

1. **Components**: 3-5 logical components derived from FRs
2. **Patterns**: 4-6 architectural patterns justified by NFRs
3. **Technologies**: Aligned with existing system architecture
4. **Metadata**: SRS reference, timestamp, validated flag
5. **FR/NFR References**: Each component/pattern must reference source requirements

### MUST Exclude

- ❌ Basic Design sections (generated by other agents)
- ❌ Implementation code
- ❌ Components not in SRS FRs
- ❌ Patterns not justified by SRS NFRs
- ❌ Technologies not in existing system architecture

---

## Template Logic (Pseudo-Code)

```pseudo
# ============================================================
# TEMPLATE: 00-reasoning.md
# PURPOSE: Generate reasoning.json for Basic Design
# ============================================================

# ────────────────────────────────────────────────────────────
# STEP 1: LOAD CONTEXT
# ────────────────────────────────────────────────────────────

FUNCTION load_context(feature_code, sub_code):
    context = {}

    # Load SRS file
    srs_file = f"documents/features/{feature_code}-{sub_code}/{feature_code}-{sub_code}-srs.md"
    context.srs = read_file(srs_file)

    # Extract FRs and NFRs
    context.frs = extract_functional_requirements(context.srs)
    context.nfrs = extract_nonfunctional_requirements(context.srs)

    # Load system architecture for technology alignment
    arch_file = "documents/architecture/01-system-architecture.md"
    IF file_exists(arch_file):
        context.architecture = read_file(arch_file)
    ELSE:
        context.architecture = null

    RETURN context

# ────────────────────────────────────────────────────────────
# STEP 2: DERIVE COMPONENTS FROM FRs
# ────────────────────────────────────────────────────────────

FUNCTION derive_components(frs):
    components = []

    # Group FRs by functional area
    functional_areas = group_by_area(frs)

    # For each area, create a component
    FOR area IN functional_areas:
        component = {
            "name": to_uppercase_snake_case(area.name),  # e.g., CASTING_MANAGEMENT
            "responsibility": area.description_vi,
            "frs": area.fr_ids,
            "rationale": f"FRs {join(area.fr_ids)} require {area.capability}"
        }
        components.append(component)

    # Validate: 3-5 components
    IF components.length < 3:
        WARN("Too few components, consider combining related FRs")
    IF components.length > 5:
        WARN("Too many components, consider grouping related functions")

    RETURN components

# ────────────────────────────────────────────────────────────
# STEP 3: SELECT PATTERNS FROM NFRs
# ────────────────────────────────────────────────────────────

FUNCTION select_patterns(nfrs):
    patterns = []

    # Pattern categories with NFR mappings
    pattern_mappings = {
        "Performance": {
            "Cache-Aside": ["NFR-PERF-*"],
            "Connection Pooling": ["NFR-PERF-*"],
            "Pagination": ["NFR-PERF-*"]
        },
        "Security": {
            "JWT Authentication": ["NFR-SEC-*"],
            "Row-Level Security": ["NFR-SEC-*"],
            "Audit Trail": ["NFR-SEC-*"]
        },
        "Reliability": {
            "ACID Transactions": ["NFR-REL-*"],
            "Optimistic Locking": ["NFR-REL-*"],
            "Retry Mechanisms": ["NFR-REL-*"]
        },
        "Integration": {
            "Event-Driven Architecture": ["NFR-INT-*"],
            "SignalR Real-time": ["NFR-INT-*"]
        },
        "Architecture": {
            "Clean Architecture": ["NFR-*"],
            "DDD (Domain-Driven Design)": ["NFR-*"]
        }
    }

    # For each NFR, select relevant patterns
    FOR nfr IN nfrs:
        matching_patterns = find_patterns_for_nfr(nfr, pattern_mappings)
        FOR pattern IN matching_patterns:
            IF NOT already_added(patterns, pattern):
                patterns.append({
                    "name": pattern.name,
                    "category": pattern.category,
                    "nfrs": [nfr.id],
                    "rationale": f"{nfr.id} requires {pattern.name}"
                })
            ELSE:
                # Add NFR to existing pattern
                existing = find_pattern(patterns, pattern.name)
                existing.nfrs.append(nfr.id)

    # Validate: 4-6 patterns
    IF patterns.length < 4:
        WARN("Too few patterns, review NFRs for additional needs")
    IF patterns.length > 6:
        WARN("Too many patterns, prioritize essential ones")

    RETURN patterns

# ────────────────────────────────────────────────────────────
# STEP 4: ALIGN TECHNOLOGIES WITH ARCHITECTURE
# ────────────────────────────────────────────────────────────

FUNCTION align_technologies(patterns, architecture):
    technologies = []

    # Map patterns to technologies
    tech_mappings = {
        "Cache-Aside": {"name": "SQL Server Cache", "purpose": "Caching for performance"},
        "ACID Transactions": {"name": "SQL Server 2022", "purpose": "Transactional integrity"},
        "Event-Driven Architecture": {"name": "SignalR", "purpose": "Real-time events"},
        "Background Jobs": {"name": "Hangfire", "purpose": "Background processing"},
        "JWT Authentication": {"name": "ASP.NET Core Identity", "purpose": "Authentication"}
    }

    FOR pattern IN patterns:
        IF pattern.name IN tech_mappings:
            tech = tech_mappings[pattern.name]
            technologies.append({
                "name": tech.name,
                "purpose": tech.purpose,
                "nfrJustification": f"{join(pattern.nfrs)} requires {tech.name}"
            })

    # Deduplicate technologies
    technologies = deduplicate(technologies)

    RETURN technologies

# ────────────────────────────────────────────────────────────
# STEP 5: GENERATE OUTPUT
# ────────────────────────────────────────────────────────────

FUNCTION generate_reasoning_json(feature_code, sub_code):
    context = load_context(feature_code, sub_code)

    components = derive_components(context.frs)
    patterns = select_patterns(context.nfrs)
    technologies = align_technologies(patterns, context.architecture)

    reasoning = {
        "components": components,
        "patterns": patterns,
        "technologies": technologies,
        "metadata": {
            "srsReference": f"{feature_code}-{sub_code}-srs.md",
            "timestamp": current_timestamp(),
            "validated": False
        }
    }

    RETURN json.stringify(reasoning, indent=2)

# ────────────────────────────────────────────────────────────
# STEP 6: VALIDATE OUTPUT
# ────────────────────────────────────────────────────────────

FUNCTION validate_output(reasoning):
    issues = []

    # Components validation
    IF reasoning.components.length < 3 OR reasoning.components.length > 5:
        issues.append(f"Component count out of range: {reasoning.components.length} (need 3-5)")

    FOR component IN reasoning.components:
        IF component.frs.length == 0:
            issues.append(f"{component.name} has no FR references")

    # Patterns validation
    IF reasoning.patterns.length < 4 OR reasoning.patterns.length > 6:
        issues.append(f"Pattern count out of range: {reasoning.patterns.length} (need 4-6)")

    FOR pattern IN reasoning.patterns:
        IF pattern.nfrs.length == 0:
            issues.append(f"{pattern.name} has no NFR references")

    # Technologies validation
    FOR tech IN reasoning.technologies:
        IF NOT tech.nfrJustification:
            issues.append(f"{tech.name} has no NFR justification")

    IF issues.length > 0:
        RETURN {"valid": False, "issues": issues}
    ELSE:
        RETURN {"valid": True, "issues": []}
```
