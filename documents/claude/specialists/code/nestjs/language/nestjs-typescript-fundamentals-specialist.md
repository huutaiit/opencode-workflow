## Patterns

### Pattern 245.1: Type-Safe Interfaces for Clean Architecture

**Category**: Type System
**Description**: Interface-based contracts between architectural layers.

```typescript
// Domain port — pure TypeScript interface
export interface OrderRepositoryPort {
  findById(id: string): Promise<Order | null>;
  save(order: Order): Promise<Order>;
  findByCustomer(customerId: string, options?: PaginationOptions): Promise<Order[]>;
}

// Application port — use case interface
export interface CreateOrderUseCase {
  execute(input: CreateOrderInput): Promise<CreateOrderOutput>;
}

// Infrastructure implements domain port
export class TypeOrmOrderRepository implements OrderRepositoryPort {
  async findById(id: string): Promise<Order | null> { /* ... */ }
  async save(order: Order): Promise<Order> { /* ... */ }
  async findByCustomer(customerId: string, options?: PaginationOptions): Promise<Order[]> { /* ... */ }
}
```

**Key Points**:
- Domain interfaces define contracts — implementations in Infrastructure
- TypeScript interfaces are erased at runtime — use Symbol tokens for DI
- `| null` explicit return — no `undefined` from repository methods

### Pattern 245.2: Decorators & Reflect Metadata

**Category**: Decorator System
**Description**: How NestJS decorators work under the hood.

```typescript
// Class decorator — @Controller, @Injectable, @Module
function Controller(prefix: string): ClassDecorator {
  return (target: Function) => {
    Reflect.defineMetadata('path', prefix, target);
    Reflect.defineMetadata('__injectable__', true, target);
  };
}

// Method decorator — @Get, @Post
function Get(path: string = ''): MethodDecorator {
  return (target, key, descriptor) => {
    Reflect.defineMetadata('method', 'GET', target, key);
    Reflect.defineMetadata('path', path, target, key);
  };
}

// Parameter decorator — @Body, @Param, @Inject
function Inject(token: any): ParameterDecorator {
  return (target, key, index) => {
    const existing = Reflect.getMetadata('self:paramtypes', target, key) || [];
    existing[index] = token;
    Reflect.defineMetadata('self:paramtypes', existing, target, key);
  };
}
```

**Key Points**:
- NestJS relies on `reflect-metadata` and `emitDecoratorMetadata: true` in tsconfig
- `design:paramtypes` auto-generated by TS compiler — provides constructor param types
- Custom decorators compose via `applyDecorators()` (NestJS utility)

### Pattern 245.3: Async/Await Patterns

**Category**: Async
**Description**: Proper async patterns for NestJS services.

```typescript
// Sequential — when operations depend on each other
async createLoan(dto: CreateLoanDto): Promise<Loan> {
  const borrower = await this.borrowerRepo.findById(dto.borrowerId);
  if (!borrower) throw new EntityNotFoundException('Borrower', dto.borrowerId);
  const risk = await this.riskService.assess(borrower);
  return this.loanRepo.save(Loan.create(dto, risk));
}

// Parallel — when operations are independent
async getDashboard(userId: string): Promise<DashboardDto> {
  const [loans, payments, notifications] = await Promise.all([
    this.loanService.findByUser(userId),
    this.paymentService.findByUser(userId),
    this.notificationService.findUnread(userId),
  ]);
  return { loans, payments, notifications };
}

// Error handling — catch specific, re-throw as domain
async processPayment(dto: PaymentDto): Promise<Payment> {
  try {
    return await this.gateway.charge(dto);
  } catch (error) {
    if (error.code === 'CARD_DECLINED') {
      throw new PaymentDeclinedException(dto.orderId, error.message);
    }
    throw new PaymentGatewayException(error.message);
  }
}
```

### Pattern 245.5: Type Guards & Discriminated Unions

**Category**: Type System
**Description**: Runtime type narrowing for domain logic.

```typescript
// Discriminated union for loan status
type LoanState =
  | { status: 'PENDING'; appliedAt: Date }
  | { status: 'APPROVED'; approvedAt: Date; interestRate: number }
  | { status: 'DISBURSED'; disbursedAt: Date; amount: number }
  | { status: 'REJECTED'; rejectedAt: Date; reason: string };

// Type guard
function isApproved(state: LoanState): state is Extract<LoanState, { status: 'APPROVED' }> {
  return state.status === 'APPROVED';
}

// Usage — TypeScript narrows type after guard
if (isApproved(loan.state)) {
  console.log(loan.state.interestRate); // TS knows interestRate exists
}
```

## Best Practices

### Type Safety
- `strict: true` in tsconfig — enables all strict type checks
- Never use `any` — use `unknown` for truly unknown types, then narrow
- Prefer interfaces for contracts, types for unions/intersections

### NestJS-Specific
- `emitDecoratorMetadata: true` required — NestJS DI relies on it
- `experimentalDecorators: true` required for decorator syntax
- `esModuleInterop: true` for CommonJS module compatibility

### Async
- Always `await` promises in services — never fire-and-forget without reason
- Use `Promise.all()` for independent parallel operations
- Use `Promise.allSettled()` when partial failure is acceptable

## Abnormal Case Patterns

1. **Runtime type error** — Interface erased at runtime, DI can't resolve. Fix: use class or Symbol token.
2. **Missing decorator metadata** — `emitDecoratorMetadata: false`. Fix: enable in tsconfig.json.
3. **Promise not awaited** — Async method called without await. Fix: ESLint rule `no-floating-promises`.
4. **Generic constraint too loose** — `T extends any` provides no type safety. Fix: constrain with `T extends BaseEntity`.
5. **Type assertion hides bug** — `as OrderDto` skips validation. Fix: validate then narrow with type guard.
